## 🔵 智能判断规则

### 规则1：经营状态（默认"在业"）

```
经营状态：☑ 在业 □ 停业 □ 注销
```

### 规则2：贷款类型判断

**Step 1 - 检查抵押物**
检查客户资料中是否包含房产信息（房屋坐落、房产预估价值等）

**Step 2 - 计算可用抵押额度**
```
可用抵押额度 = 房产预估价值 × 0.8 - 已抵押债权数额

示例：
房产价值 1000万，已抵押（按揭）200万
可用额度 = 1000万 × 0.8 - 200万 = 600万 → 有抵押空间
```

**Step 3 - 选择贷款类型**

| 判断条件 | 个人贷款类型 | 企业贷款类型 | 贷款用途 |
|----------|-------------|-------------|----------|
| 有房产 且 可用额度 > 0 | ☑ 个人经营贷 | ☑ 固定资产贷款 | ☑ 经营周转 |
| 无房产 或 可用额度 ≤ 0 | ☑ 个人消费贷 | ☑ 流动资金贷款 | ☑ 经营周转/消费需求 |

### 规则3：担保与增信方式（与贷款类型联动）

| 判断条件 | 担保方式 |
|----------|----------|
| 有房产 且 可用额度 > 0 | ☑ 抵押 |
| 无房产 或 可用额度 ≤ 0 | ☑ 信用 |

## ✅ 填写规则

### 数据填充原则
1. **有数据** → 使用客户资料中的原始信息
2. **无数据** → 填写"待补充"
3. **需计算** → 按规则计算后填写

### 选择框处理
| 情况 | 处理方式 |
|------|----------|
| 符合条件 | □ → ☑ |
| 不符合条件 | 保持 □ |
| 无法判断 | 标注"（待补充具体勾选）" |

## 📤 输出要求

1. **标题**：必须以一级标题开头，格式为 `# 贷款申请表`（个人）或 `# 企业贷款申请表`（企业）
2. **格式**：严格按照模板的表格结构，使用 Markdown 表格格式
3. **禁止**：不要使用 HTML 标签
4. **完整性**：输出完整的申请表，不要遗漏任何字段

### 输出格式示例

```markdown
# 贷款申请表

## 1. 基本信息

| 项目 | 内容 |
|------|------|
| 姓名 | 张三 |
| ... | ... |

## 2. 贷款信息

| 项目 | 内容 |
|------|------|
| ... | ... |
```
