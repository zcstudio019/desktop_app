## 1. 项目概述

### 1.1 背景
内部员工使用的贷款申请辅助工具，用于：
- 上传客户资料并提取关键信息
- 根据客户资料生成贷款申请表
- 根据客户资料匹配合适的贷款产品

### 1.2 技术栈
| 组件 | 选择 |
|------|------|
| 前端框架 | React 18 + Vite + TypeScript |
| 后端框架 | FastAPI + Uvicorn |
| UI 框架 | Tailwind CSS v3.4.1 + Lucide Icons |
| AI | DeepSeek API |
| OCR | 百度 OCR |
| 存储 | 飞书多维表格 |

### 1.3 用户
- 内部员工自用
- 无需登录认证

### 1.4 页面结构
| 页面 | 组件 | 说明 |
|------|------|------|
| 仪表盘 | DashboardPage | 统计数据和最近活动 |
| 上传资料 | UploadPage | 文件上传和 AI 提取 |
| 申请表 | ApplicationPage | 贷款申请表生成 |
| 方案匹配 | SchemeMatchPage | 贷款产品匹配 |
| AI 对话 | ChatPage | 智能对话（集成提取/申请/匹配） |

## 2. 功能模块

### 2.1 上传资料

#### 2.1.1 功能描述
用户上传客户资料文件，系统自动提取关键信息并保存到飞书多维表格。

#### 2.1.2 支持的资料类型
| 资料类型 | 支持格式 | 提示词文件 |
|---------|---------|-----------|
| 企业征信 | PDF、图片 | `提示词/企业征信提取_提示词.md` |
| 个人征信 | PDF、图片 | `提示词/个人征信提取_提示词.md` |
| 企业流水 | Excel、PDF | `提示词/企业流水提取_提示词.md` |
| 个人流水 | Excel、PDF | `提示词/个人流水提取_提示词.md` |
| 财务数据 | Excel | `提示词/财务数据提取_提示词.md` |
| 抵押物信息 | 图片 | `提示词/抵押物信息提取提示词.md` |
| 水母报告 | PDF | `提示词/水母报告提取提示词.md` |
| 个人纳税/公积金 | PDF、图片 | `提示词/个人纳税/公积金提示词.md` |

#### 2.1.3 用户操作流程
```
1. 在侧边栏选择「上传资料」页面
2. 选择资料类型（卡片网格选择）
3. 拖拽或点击上传文件（支持批量）
4. 可选：填写客户名称
5. 系统自动处理并显示提取结果
6. 自动保存到飞书（或手动点击保存）
```

#### 2.1.4 核心需求

**REQ-2.1.1 文件上传**
- 支持批量上传多个文件
- 根据资料类型限制可上传的文件格式
- 文件大小限制：10MB
- 图片超过 4MB 自动压缩（百度 OCR 限制）

**REQ-2.1.2 文件处理**
| 文件类型 | 处理方式 |
|---------|---------|
| 图片 | 百度 OCR 识别文字 |
| PDF（文本型） | 直接提取文本 |
| PDF（扫描件） | 转图片后 OCR |
| Excel | 读取所有 Sheet 内容 |
| Word | 提取文本内容 |

**REQ-2.1.3 AI 提取**
- 根据资料类型加载对应提示词
- 调用 DeepSeek API 提取结构化数据
- 输出 JSON 格式

**REQ-2.1.4 智能合并（核心）**
- 根据客户名称（企业名称/个人姓名）匹配飞书表格中的已有记录
- 匹配规则：在以下字段中搜索客户名称
  - 个人/企业基本信息
  - 企业/个人征信
  - 企业/个人流水
  - 财务数据
  - 水母报告
  - 个人收入纳税/公积金

**REQ-2.1.5 追加逻辑（重要）**
- **追加字段**：同类型资料上传时，新内容追加到旧内容后面，用 `=== 最新更新 ===` 分隔
  - 个人/企业基本信息
  - 企业/个人征信
  - 企业/个人流水
  - 资金需求
  - 抵押物信息
  - 财务数据
  - 公共记录
  - 风险提示
  - 还款进度
  - 逾期提醒
  - 备注
  - 水母报告
  - 个人收入纳税/公积金

- **覆盖字段**：直接用新值覆盖旧值
  - 序号
  - 客户复贷潜力标签
  - 上传时间

**REQ-2.1.6 客户名称自动提取**
- 用户不填写客户名称时，从 AI 提取结果中自动获取
- 搜索字段优先级：
  1. 企业名称、公司名称、编制单位
  2. 权属人、借款人、申请人、账户持有人
  3. 姓名、被查询者姓名、客户姓名
  4. 嵌套结构中的对应字段
  5. 流水数据的「关联方交易」中的「关联企业名称」

**REQ-2.1.7 飞书显示格式**
- 保存到飞书时转换为 Markdown 格式（非 JSON）
- 便于在飞书表格中阅读

#### 2.1.5 界面要求
- 显示当前选择的资料类型
- 显示上传文件的名称、大小、类型
- 提取过程显示进度（spinner）
- 批量上传时显示进度条
- 提取结果可展开/折叠查看
- 支持下载 JSON 结果
- 保存成功显示记录 ID
- 保存失败显示错误信息

#### 2.1.6 错误处理
| 错误类型 | 处理方式 |
|---------|---------|
| 文件过大 | 提示压缩后重新上传 |
| 格式不支持 | 显示允许的格式列表 |
| OCR 失败 | 显示错误原因，提供重试按钮 |
| AI 超时 | 提示稍后重试 |
| 飞书认证失败 | 提示检查配置 |
| 网络错误 | 提示检查网络连接 |

#### 2.2.5 填写助手提示词

**提示词文件**：`提示词/贷款申请表填写助手_提示词.md`

**输入变量**
| 变量名 | 说明 | 来源 |
|-------|------|------|
| `{{customer_data}}` | 个人客户资料 | 飞书多维表格 |
| `{{customer_data_1}}` | 企业客户资料 | 飞书多维表格 |
| `{{template_1}}` | 个人贷款模板 | 模板文件 |
| `{{template}}` | 企业贷款模板 | 模板文件 |

**核心规则**
1. **禁止编造字段**：期望额度、期望期限、利率、贷款金额必须填写「待补充」
2. **经营状态默认**：默认填写「在业」
3. **贷款类型判断**：根据房产信息和可用抵押额度自动判断
   - 有房产且可用额度 > 0 → 抵押贷/固定资产贷
   - 无房产或可用额度 ≤ 0 → 信用贷/流动资金贷
4. **担保方式联动**：与贷款类型保持一致
5. **数据填充原则**：有数据用原始信息，无数据填「待补充」

#### 2.2.6 核心需求

**REQ-2.2.1 用户输入**
- 客户名称：文本输入框（必填）
- 贷款类型：下拉选择（企业贷款 / 个人贷款）

**REQ-2.2.2 客户资料获取**
- 从飞书多维表格搜索客户记录
- 使用 `feishu_service.search_records(customer_name)`
- 找不到客户时提示「未找到该客户的资料，请先上传客户资料」

**REQ-2.2.3 模板加载**
- 根据贷款类型加载对应模板
- 模板文件位于 `提示词/` 目录
- 使用 FastAPI 服务端缓存（`lru_cache`）缓存模板内容

**REQ-2.2.4 AI 填充**
- 调用 DeepSeek API
- 使用 `贷款申请表填写助手_提示词.md`
- 替换提示词中的变量后调用
- temperature=0.1（确保填充结果一致性）

**REQ-2.2.5 输出格式**
- 主要格式：Markdown（直接在页面渲染显示）
- 下载格式：
  - Markdown (.md)
  - Word (.docx) - 使用 python-docx 转换

**征信类型与产品库映射**
| 征信类型 | 获取的产品库 | 传入提示词变量 |
|---------|-------------|---------------|
| 个人征信 | 个人贷款产品库 | `loan_products_2` |
| 企业征信 | 企业信用贷 + 企业抵押贷（合并） | `loan_products_1` |

**匹配流程**
```
1. 提取客户关键信息
   - 企业客户：成立年限、纳税等级、年纳税额、负债率、房产情况等
   - 个人客户：年龄、职业、收入、房产、征信情况等

2. 判断房产抵押状态（企业客户）
   - 房产已抵押 → 仅匹配信用贷
   - 房产未抵押 → 匹配抵押贷 + 信用贷
   - 无房产 → 仅匹配信用贷

3. 逐产品核对准入条件
   - 遍历产品库中每个产品
   - 逐条核对准入条件
   - 全部满足才推荐

4. 按优先级排序
   - 利率最低优先
   - 额度最高优先
   - 期限最长优先
```

#### 2.3.6 提示词

**提示词文件**：`提示词/方案匹配智能体提示词.md`

**输入变量**
| 变量名 | 说明 | 来源 |
|-------|------|------|
| `{{customer_info}}` | 客户资料 | 飞书多维表格 |
| `{{loan_products_1}}` | 企业贷款产品库 | 飞书云文档（企业信用贷 + 企业抵押贷） |
| `{{loan_products_2}}` | 个人贷款产品库 | 飞书云文档（个人贷款） |

**核心规则**
1. 产品名称、利率、额度、期限必须 100% 来自产品库原文
2. 禁止编造任何产品信息
3. 能匹配就匹配，不要反复追问
4. 先给方案，再说"补充 xx 信息可以匹配更多产品"

## 5. UI 美化需求

> ⚠️ 本节内容已不适用。项目已从 Streamlit 迁移到 React + Tailwind CSS 架构。
> 当前 UI 使用 Tailwind CSS v3.4.1 + Lucide Icons，无需 streamlit-shadcn-ui。

### 5.1 背景

当前 Streamlit 默认界面较为朴素，为提升用户体验和专业感，计划使用 `streamlit-shadcn-ui` 组件库进行界面美化。

### 5.2 技术方案

**选用组件库**：[streamlit-shadcn-ui](https://github.com/ObservedObserver/streamlit-shadcn-ui)

| 项目 | 说明 |
|------|------|
| 组件库 | streamlit-shadcn-ui |
| 设计风格 | Shadcn UI（现代、简洁、专业） |
| 安装方式 | `pip install streamlit-shadcn-ui` |
| 官方演示 | https://shadcn.streamlit.app/ |

**选择理由**：
- 直接可用，无需大改现有代码
- 现代化 Shadcn 风格，专业感强
- 支持 Cards、Tables、Badges、Buttons 等常用组件
- 社区活跃，持续维护

- 移动端基本可用
